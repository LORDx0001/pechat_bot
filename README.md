# Telegram Print Ordering Bot & Backend API (SQLite3)

Этот проект представляет собой Telegram-бот для заказа одежды с нанесением принтов. Работает на локальной базе SQLite3 и стандартных сервисах без Docker.

## Архитектура системы

- **Backend API**: Django 5 + Django REST Framework + Django Admin
- **Database**: SQLite3 (локальный файл `db.sqlite3`)
- **Telegram Bot**: Aiogram 3 (Python 3.13)
- **Уведомления**: Отправляются асинхронно с помощью фоновых потоков (`threading`) в самом Python (без Redis и Celery)

## Быстрый запуск

### 1. Подготовка окружения
Создайте и активируйте виртуальное окружение Python:
```bash
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
# или venv\Scripts\activate для Windows
```

### 2. Установка зависимостей
Установите библиотеки для Django-сервера и Telegram-бота:
```bash
pip install -r backend/requirements.txt
pip install -r bot/requirements.txt
```

### 3. Переменные окружения
Скопируйте файл конфигурации окружения:
```bash
cp .env.example .env
```
Откройте файл `.env` и вставьте ваш токен Telegram-бота:
```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота_здесь
```

### 4. Запуск Django Backend
В первом терминале с активированным виртуальным окружением выполните:
```bash
# Примените миграции базы данных (создастся файл db.sqlite3)
python backend/manage.py migrate

# Наполните базу данных демонстрационными данными (товары, размеры, цвета, реквизиты)
python backend/manage.py seed_data

# Создайте суперпользователя для панели управления (Django Admin)
python backend/manage.py createsuperuser

# Запустите локальный сервер Django
python backend/manage.py runserver
```
Сервер будет доступен по адресу **`http://127.0.0.1:8000/`**.

### 5. Запуск Telegram-бота
Во втором терминале (также с активированным виртуальным окружением и настроенным `.env`) запустите бота:
```bash
python bot/main.py
```

## Настройка панели администратора

Откройте панель администратора: **`http://127.0.0.1:8000/admin/`**

1. **Настройки магазина (StoreSettings)** (Singleton-модель):
   - Настройте **Название магазина** (Shop name), **Текст приветствия** (Welcome text) и **Контакты** (Contacts).
   - Укажите **Telegram менеджера** (например: `@manager_username`) для раздела поддержки.
   - **ВАЖНО**: Укажите **Telegram ID чата уведомлений** (`admin_chat_id`). Это ваш личный Telegram ID или ID группы администраторов. Бот будет отправлять туда уведомления о новых заказах и чеках на проверку. (Получить свой ID можно у бота `@userinfobot`).
2. **Размеры (Sizes) и Цвета (Colors)**:
   - Добавьте размеры (`S`, `M`, `L`, `XL` и т.д.).
   - Добавьте цвета (например: `Черный` с hex `#000000`, `Белый` с hex `#FFFFFF`).
3. **Места нанесения (PrintPositions)**:
   - Добавьте варианты (например: `Спереди` с наценкой `0`, `Сзади` с наценкой `150`).
4. **Способы оплаты (PaymentMethods)**:
   - Добавьте реквизиты (Карта Сбербанка, Получатель). Обязательно отметьте как **Активен** (Active).
5. **Категории (Categories) и Товары (Products)**:
   - Добавьте категории (`Футболки`, `Худи`) и товары под ними. Свяжите каждый товар с доступными цветами и размерами.

## Функционал Telegram-бота

1. **Старт (`/start`)**: Регистрация пользователя в БД и вывод приветствия.
2. **Заказ (`🛍 Сделать заказ`)**: FSM-поток выбора категории ➡️ товара ➡️ размера ➡️ цвета ➡️ места печати ➡️ загрузки макета (документом/фото) ➡️ комментария ➡️ количества.
3. **Корзина (`🛒 Корзина`)**: Просмотр состава корзины, редактирование количеств (кнопками `➕` / `➖`), удаление, оформление заказа.
4. **Оформление**: Сбор контактов (имя, телефон, адрес доставки) и создание заказа со статусом `WAITING_PAYMENT`.
5. **Оплата**: Показ реквизитов карт. По кнопке «Я оплатил» пользователь загружает фото чека (статус заказа меняется на `RECEIPT_PENDING`).
6. **Проверка чека**: Бот отправляет чек в административный чат с кнопками **✅ Подтвердить** и **❌ Отклонить**, которые мгновенно обновляют базу данных и уведомляют пользователя о результате.
